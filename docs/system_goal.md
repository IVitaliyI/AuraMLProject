# System Goals and Boundaries

## System Goal

Система предназначена для **разработчика и аналитика**,
чтобы **получать релевантные запросу документы на основе собственной базы документов**,
путём **REST API и Web-интерфейса**,
при ограничениях **self-hosted развертывания, контролируемого доступа к данным
и отсутствия автоматического обучения моделей**.

---

## Stakeholders and Actors

| Actor        | Role                         | Goal                                  |
|-------------|------------------------------|---------------------------------------|
| End User    | задаёт вопросы               | получает релевантные документы |
| Developer   | настраивает и расширяет систему | гибкая интеграция и контроль           |
| External LLM Client | внешнее приложение | использует результаты retrieval |

---

## In Scope

Система:
- принимает документы для индексирования (PDF, TXT, MD)
- извлекает текст и метаданные
- строит и хранит эмбеддинги
- выполняет семантический поиск
- возвращает ранжированный список документов или текстовых фрагментов
  с метаданными и идентификаторами источников
---

## Out of Scope

Система **не**:
- обучает или дообучает LLM
- управляет биллингом пользователей
- предоставляет полноценную IAM-платформу
- гарантирует фактическую истинность полученных документов

## Success Criteria

- Добавление нового источника документов без изменения core-логики
- Повторяемость результатов поиска при одинаковых входных данных
- Возможность полного переиндексирования системы
